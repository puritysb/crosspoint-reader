# Korean Font Setup (RIDIBatang)

CrossPoint's built-in fonts (Noto Serif / Noto Sans) only carry Latin glyphs —
Korean text renders as tofu (□) on a stock device. This guide installs
**RIDIBatang** (리디바탕, RIDI Corp, SIL OFL) as an SD card font so EPUBs with
Korean (translated novels, original Korean works) render cleanly on the X3/X4
e-ink panel. RIDIBatang is widely regarded as one of the most readable Korean
serifs on e-ink.

## What gets installed

Four `.cpfont` files covering sizes 12 / 14 / 16 / 18 pt, all regular style
(RIDIBatang ships regular only; bold/italic fall back to regular, which is the
norm for Korean body text):

```
.fonts/RIDIBatang/RIDIBatang_12.cpfont   (~1.6 MB)
.fonts/RIDIBatang/RIDIBatang_14.cpfont   (~2.2 MB)
.fonts/RIDIBatang/RIDIBatang_16.cpfont   (~2.8 MB)
.fonts/RIDIBatang/RIDIBatang_18.cpfont   (~3.5 MB)
```

Each file bundles ~11,940 glyphs across the `reading` preset (Latin, Greek,
Cyrillic, punctuation, symbols, etc.) plus the `hangul` preset (Hangul Syllables
U+AC00–U+D7AF, Jamo U+1100–U+11FF, Compatibility Jamo U+3130–U+318F). One font
family covers every script the reader is likely to encounter.

## Source

- Font: <https://cdn.jsdelivr.net/gh/fonts-archive/RIDIBatang/RIDIBatang.otf>
- License: SIL Open Font License 1.1 (저작권 — 리디주식회사)
- Conversion: `lib/EpdFont/scripts/fontconvert_sdcard.py` with `--intervals reading,hangul`

The family is also registered in `lib/EpdFont/scripts/sd-fonts.yaml`, so running
`build-sd-fonts.py` regenerates the four files from source on demand.

## Installation (UI path — recommended)

1. Copy the four `.cpfont` files into `/SD/.fonts/RIDIBatang/` (create the
   folder if missing). macOSAppleDouble files (`._*`) are auto-ignored by
   `SdCardFontRegistry::scanDirectory`.
2. Eject the SD, insert into the X3 / X4, boot.
3. **Settings → Manage Fonts** → wait for the scan to pick up `RIDIBatang`.
4. **Settings → Reader → Font Family** → select `RIDIBatang`.
5. Open any EPUB with Korean (e.g. `books/bilingual-sample.epub`). The
   `cp-translation` (Korean) paragraphs should render cleanly.

## Installation (manual, when Manage Fonts is unavailable)

If the UI doesn't list `RIDIBatang` after the scan, force a re-scan by deleting
the registry cache on the SD card:

```
/SD/.crosspoint/font_registry.bin   ← delete and reboot
```

The next boot re-scans `.fonts/` from scratch.

## Verification

Open `books/bilingual-sample.epub`, the bilingual sample generated by
`scripts/generate_bilingual_test_epub.py`. With Font Family set to `RIDIBatang`:

- **Both** mode: every paragraph (English + Korean) renders.
- **Original only**: English only (Latin glyphs from RIDIBatang).
- **Translation only**: Korean only (Hangul syllables).

If Korean still shows tofu after switching to RIDIBatang:
1. Confirm the SD path is exactly `/SD/.fonts/RIDIBatang/RIDIBatang_*.cpfont`.
2. Run `pio device monitor` (or check `agentdeck.log` on SD if USB-CDC is
   unavailable) for `SDFONT` log lines — they list families found during scan.
3. Re-generate from source with `python3 lib/EpdFont/scripts/build-sd-fonts.py
   --only RIDIBatang`.

## Rebuilding from source

```bash
cd lib/EpdFont/scripts
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
.venv/bin/python build-sd-fonts.py --only RIDIBatang
# → output/RIDIBatang/RIDIBatang_{12,14,16,18}.cpfont
```
